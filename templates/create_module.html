Type into the filter box to filter the results. You can just type stuff, but there are also some neat things you can try:

  • It's not case sensitive.
  • Name is science machine will return the object with the name science machine. The format "[field] is [value]" can be used with any field. "Tab is light", for example, will return all items under the light tab. Equals is an alias for is, so you can use either according to preference.
  • Name contains pig will return any objects that have pig in their names. The format "[field] contains [value]" can be used with any field. "Tech contains science", for example, will return all items under the science tech tree (science machine, alchemy engine).
  • Uses twigs will return any objects that require twigs to build them. This matches in the same was as contains, so "uses turf" will return items that require any turf to build them. It also checks the tech required to build it. "Uses science machine" will return all items you need the science machine to build. Requires is an alias for uses, so you can use either according to preference.
  • Uses 3 twigs will return any objects that require exactly 3 twigs to build. The format is "uses [number] [ingredient]".
  • Uses greater than 3 twigs will return any objects that require more than 3 twigs to build. The format is "uses greater than [number] [ingredient]". More is an alias for greater, so you can use either according to preference.
  • Uses less than 3 twigs will return any objects that require less than 3 twigs to build. However it will only return items that do require twigs. If you want items that don't require twigs as well, you can use "Uses less than 3 twigs and not uses twigs". The format is "uses less than [number] [ingredient]". Fewer is an alias for less, so you can use either according to preference.
  • shroom will return all objects with "shroom" contained in any field, including ingredients.
  • This tool uses Logipar, so you can combine any of the above using AND, OR, NOT, XOR, and ( parentheses ) to your heart's content.

The field list is:

  • Name - the object's name in the source files.
  • Friendlyname - the object's name in-game.
  • Tab - the tab the object is under.
  • Tech - the part of the tech tree required, as per the source files.
  • Friendlytech - the part of the tech tree required, as per in-game.
  • Description - the in-game description string.

A note about "friendly" fields. If you search a field that has a corresponding "friendly" field, both will be searched automatically. So searching "name contains cat" will search both the name field, and the friendlyname field.
